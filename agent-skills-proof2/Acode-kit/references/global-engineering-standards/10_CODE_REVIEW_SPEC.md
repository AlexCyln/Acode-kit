# 文件名
`10_CODE_REVIEW_SPEC.md`

# 文件定位
Code Review 制度与检查标准。

# 适用范围
适用于前端、后端、数据库脚本、配置变更、部署脚本、测试脚本等代码交付物。

# 与其他文件的关系
本文件基于 `08_CODE_STYLE_AND_NAMING_SPEC.md`、`03_FRONTEND_ARCHITECTURE_SPEC.md`、`04_BACKEND_ARCHITECTURE_SPEC.md`、`16_SECURITY_SPEC.md` 执行审查。

# 编写目的
将 Code Review 从个人习惯升级为制度化质量门禁，统一审查维度、结论等级与整改标准。

## 1. Review 目标
1. 发现功能缺陷与回归风险。
2. 发现安全、性能、可维护性问题。
3. 保证实现与需求、设计、接口契约一致。
4. 促进知识共享与规范落地。

## 2. Review 必查项
1. 是否满足需求与验收标准。
2. 是否遵守架构分层与命名规范。
3. 是否覆盖异常、边界、权限、幂等场景。
4. 是否带来安全、性能、数据一致性风险。
5. 是否补充必要测试。

## 3. 前端 Review 重点
1. 页面结构是否符合设计稿和页面范式。
2. 业务逻辑是否从视图层拆出。
3. 路由、菜单、权限是否统一。
4. 请求、错误处理、表单校验是否完整。
5. 是否存在超大组件、重复逻辑、硬编码字典。
6. 是否对 `localStorage`、`sessionStorage` 中的登录态和展示字段做了脏数据兼容、版本迁移或坏值清理。

## 4. 后端 Review 重点
1. Controller、Service、Mapper 职责是否越界。
2. DTO、VO、Entity 是否混用。
3. 事务边界是否合理。
4. 参数校验、异常处理、日志审计是否完整。
5. 是否存在 N+1 查询、无索引查询、无幂等保护。
6. 当前后端分离部署时，是否显式覆盖浏览器 CORS 预检场景，而不只是 `curl` 调通。

## 5. 数据库 Review 重点
1. 表结构是否符合建模规则。
2. 命名、主键、审计字段、逻辑删除策略是否统一。
3. 索引是否匹配查询场景。
4. DDL 是否兼顾回滚与历史数据兼容。

## 6. API Review 重点
1. URI、Method 是否语义正确。
2. 字段命名是否统一。
3. 错误码、分页、批量、导入导出是否规范。
4. 接口是否声明权限、幂等、版本要求。
5. 对于浏览器调用的接口，是否验证过 `OPTIONS` 预检、跨域响应头和真实登录链路。

## 7. 安全 Review 重点
1. 是否存在越权访问风险。
2. 是否校验输入合法性。
3. 是否泄露敏感信息。
4. 是否具备登录态、Token、审计、防重放控制。

## 8. 性能 Review 重点
1. 是否存在明显高复杂度循环或大查询问题。
2. 列表接口是否分页。
3. 缓存使用是否合理。
4. 大文件导出导入是否异步化。

## 9. 可维护性 Review 重点
1. 模块边界是否清晰。
2. 是否有重复实现。
3. 是否有可读性差的超长方法或条件分支。
4. 是否保留足够注释与测试上下文。

## 10. Review 结论等级
1. `Approve`：可合并。
2. `Approve with minor comments`：有小问题，可在合并前或合并后立即修复。
3. `Request changes`：存在功能、架构、安全、可维护性问题，必须整改后再审。
4. `Block`：存在严重风险，禁止进入主线。

## 11. 驳回与整改标准
以下问题默认驳回：
1. 与需求或设计明显不一致。
2. 绕过权限、鉴权、校验、审计。
3. 引入高风险 SQL、缓存一致性问题、事务问题。
4. 缺少关键测试与回归说明。

## 12. Review 评论写法
1. 指出问题事实。
2. 说明风险或可能后果。
3. 给出建议方向。
4. 引用相关规范文件。

示例：
```md
这里直接在 Controller 调用 Mapper，违反 `04_BACKEND_ARCHITECTURE_SPEC.md` 的分层边界。
风险是事务和业务校验被绕过，建议改为通过 Service 编排。
```

## 13. Review 清单模板

```md
- 架构边界：
- 权限与安全：
- 数据一致性：
- 跨域与环境配置：

- 测试覆盖：
- 风险说明：
- 结论：
```

## 本文件使用建议
将本文件固化为 PR 模板和团队 Review 清单，确保每次审查至少覆盖功能、架构、安全、测试四个维度。

## AI 调用建议
AI 在执行代码审查时，应以本文件的检查项和结论等级输出审查意见，优先指出实质风险而非样式噪音。
