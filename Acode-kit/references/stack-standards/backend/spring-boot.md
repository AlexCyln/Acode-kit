# 文件定位
Spring Boot 后端实现规范包。

## 适用范围
- Java 后端服务
- 管理后台 / 业务平台 / API 平台
- 单体优先、模块化清晰的企业级后端项目

## 典型技术组合位置
- Spring Boot 作为主服务框架
- 与 MyBatis-Plus、Redis、消息或第三方接口协作
- 默认与 Flyway 或 Liquibase 协同完成 schema 版本治理

## 推荐目录结构
- `controller` / `application` / `domain` / `infrastructure` 分层清楚
- `config`、`security`、`integration`、`job`、`support` 等横切模块边界清晰
- DTO、VO、Entity、Assembler/Convertor 职责分离

## 分层与职责边界
- Controller 仅做协议适配、参数接收、鉴权入口和响应转换
- Application 负责用例编排、事务边界、权限协调和审计触发
- Domain 负责核心业务规则、状态机和领域校验
- Infrastructure 负责数据库、缓存、消息、第三方接口等技术实现
- 不允许在 Controller、Filter、Interceptor、Mapper 中偷塞核心业务规则

## 编码复杂度与可维护性要求
- Service / Application 层若出现超过 2-3 层控制流嵌套，必须评估 early return、规则对象、状态机或用例拆分
- 审批流、状态流、计费规则优先显式建模，不写成长条件链
- 超长方法通常意味着职责过重，应拆为校验、装配、状态变更、持久化、审计、通知等可测试步骤

## 测试策略与质量门禁
- Service 逻辑优先单元测试
- 关键 API 做集成测试
- 状态流、权限流、事务边界优先覆盖
- Migration 脚本必须有迁移前后验证点
- 多租户项目必须有跨租户越权回归用例
- 关键导出、审批、回调、幂等接口必须有失败路径测试

## 数据与 Migration 协同要求
- 默认使用 Flyway；若项目声明 Liquibase，必须写入 `ACTIVE_STANDARDS.md`
- Migration 目录、命名、执行模式必须项目级统一
- 结构变更、索引变更、约束变更统一走 Migration，不得手工改正式环境库
- 大表索引、非空约束、历史回填要拆分评审并说明前向兼容策略
- 不在 `CommandLineRunner`、启动钩子或零散脚本中偷偷修结构或回填数据

## API / 安全 / 权限协同要求
- API 版本策略必须显式声明
- 高风险接口必须说明幂等、权限、审计和限流要求
- 前端展示权限与后端最终鉴权必须同时存在
- 平台级权限、租户级权限、组织级数据权限不得混写

## 日志与可观测性协同要求
- 统一记录 `traceId`、`tenantId`、`userId`、模块和动作语义
- 提供健康检查与运行指标端点
- 关键业务链路必须有业务日志与核心指标
- 发布后异常率、慢接口、数据库连接池和核心任务积压必须可观测

## 多租户适配要求
- 多租户项目必须在入口层明确租户上下文来源
- Service、Repository、缓存键、日志字段都必须体现租户边界
- 不允许只在 Controller 层拼接 `tenantId` 过滤

## 文档要求
- API 文档、模块文档、数据库文档同步维护
- Migration 策略、租户模式、权限模型、审计字段和观测指标要写入开发文档

## 常见误用
- Controller 写业务
- Mapper 直接承载业务规则
- 把异常、权限、日志、租户过滤写成每个模块各自一套
- 直接手工改库后再补 Java 代码

## 禁止事项
- 在接口和领域模型未冻结时直接展开大规模编码
- 在未声明 Migration 策略和租户模式前扩展大量持久化实现

## 追加增强：企业级实现细则

### 用例编排细则
- 一个用例应有单一入口方法，不允许同一业务链路在多个 Service 中散乱拼接
- 用例入口方法建议按“校验 → 装配上下文 → 状态变更 → 持久化 → 审计/通知”顺序组织
- 外部副作用优先后置，必要时通过事件或任务解耦

### 事务与一致性细则
- 事务内只保留必须原子化的步骤
- 跨系统调用、文件 IO、长耗时任务不要放在主事务内
- 幂等、重试、补偿、审计要在设计时就有明确落点

### 安全与权限细则
- 接口权限、数据权限、租户权限三者必须拆开思考
- 高风险操作必须带审计语义和失败回执

### 评审附加清单
- 是否存在超长 Application Service
- 是否把领域规则写进了 Mapper 或配置层
- 是否缺失 traceId / tenantId / userId 等关键上下文字段
