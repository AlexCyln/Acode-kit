# 文件定位
Vue 3 + Vite 前端实现规范包。

## 适用范围
- Web 应用
- 数据驱动后台
- 内容与配置并重的中后台项目

## 典型技术组合位置
- Vue 3 负责页面、状态和组件组合
- Vite 负责构建、开发和环境管理
- 可与 Pinia、Vue Router、Element Plus 或自定义设计体系协同

## 推荐目录结构
- `src/views/` 承载路由页面
- `src/components/` 放共享组件和页面区块
- `src/features/` 或 `src/modules/` 按业务拆分
- `src/router/`、`src/stores/`、`src/api/`、`src/composables/`、`src/utils/` 职责清晰
- `public/` 承载静态资源与外部公开文件

## 分层与职责边界
- view 负责页面编排与状态承接
- composable 负责可复用交互与数据逻辑
- store 负责全局状态，不承载页面级一次性逻辑
- api 层负责请求协议与错误适配
- 不允许把请求、权限、埋点、复杂表单全部堆在单个 SFC 中

## 编码复杂度与可维护性要求
- 组合式 API 优先，状态与副作用显式拆分
- 筛选项、表单 schema、表格列和权限控制要模块化
- 大页面优先拆成容器层与展示层
- 路由守卫、权限守卫、租户上下文守卫要统一入口

## 测试策略与质量门禁
- 关键页面与 composable 要有 Vitest 测试
- 核心路由、表单、筛选和关键操作要有浏览器级 smoke
- loading、error、empty、disabled 等状态必须覆盖

## 数据与 Migration 协同要求
- API 字段、状态字典、筛选项和导出字段要与后端保持一致
- schema 或字典变更时，view、store、composable 和测试要同步更新

## API / 安全 / 权限协同要求
- 请求通过统一 api 层，不在页面中散落直接请求
- 前端权限判断只做体验控制，后端仍是最终鉴权
- 路由权限、按钮权限、数据权限需要统一命名与来源

## 日志与可观测性协同要求
- 页面异常、接口失败、关键交互失败要可追踪
- 关键链路应记录版本、页面、用户和租户上下文

## 多租户适配要求
- 多租户项目必须明确租户切换、缓存清理和权限刷新机制
- 不允许在本地状态中长期残留跨租户脏数据

## 文档要求
- 模块文档说明 view、composable、store、api 的边界
- 测试文档记录关键表单、筛选、路由和异常态覆盖

## 常见误用
- 单个 SFC 承载过多逻辑
- store 变成全局杂物间
- 路由守卫和按钮权限各写一套

## 禁止事项
- 在页面骨架未冻结时堆叠大量复用抽象
- 在权限模型未稳定时散落写死角色判断

## 追加增强：Vue 3 + Vite 细则

### 页面组织细则
- 列表页、详情页、编辑页优先形成稳定骨架
- 公共区块通过组件或 slot 复用，不复制粘贴页面模板

### 状态细则
- 局部状态优先局部管理，全局状态只保留真正跨页面共享内容
- 路由参数、筛选参数、分页参数要显式建模

### Review 附加清单
- 是否存在 store 过度膨胀
- 是否存在页面中混合过多副作用
- 是否存在权限和请求逻辑散落
