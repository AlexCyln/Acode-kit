# 文件定位
MyBatis-Plus 数据访问规范包。

## 适用范围
- Spring Boot + MyBatis-Plus 数据访问项目
- 以关系型数据库为主存储的业务系统

## 典型技术组合位置
- Mapper 层负责持久化
- 与 Entity、Query、分页对象、Convertor 配合
- 与 Migration、数据字典、租户隔离、审计字段统一协作

## 推荐目录结构
- `mapper/`、`entity/`、`query/`、`convertor/` 分离
- XML、自定义 SQL、Wrapper 构造策略有统一落点
- 复杂查询不要塞进 Service 字符串拼接

## 分层与职责边界
- Entity 只映射持久化结构，不承载业务流程
- Query 对象显式承载筛选、分页、排序入参
- Mapper 负责查询语义，不负责业务规则编排
- Convertor 负责 Entity/VO/DTO 的边界转换

## 编码复杂度与可维护性要求
- SQL 条件构造优先可组合、可复用，不要散落隐式拼接
- 所有排序字段必须白名单控制
- Mapper 方法命名与查询语义保持一致，禁止 `selectList1`、`queryData` 这类模糊命名
- 复杂 SQL 若出现 2-3 层以上条件拼接，应评估拆成显式查询对象或独立 SQL 片段

## 测试策略与质量门禁
- 复杂查询必须有集成测试
- 排序、筛选、分页和边界数据要覆盖
- 索引依赖的高频查询必须验证 explain 或至少验证关键条件命中策略
- 回填或历史兼容查询必须验证旧数据和新数据并存场景

## 数据与 Migration 协同要求
- Mapper、Entity、XML 与 schema 变更必须同版本评审
- 新增字段、索引、约束后，Mapper 查询和写入逻辑必须同步更新
- 禁止依赖“线上表已经手工改过”这一前提写 Mapper 代码
- 在 Migration 未入库前，不得依赖新 schema 写持久化逻辑

## API / 安全 / 权限协同要求
- Query 入参与 API DTO 字段语义保持一致
- 列表、详情、导出都必须遵守后端最终数据权限
- 敏感字段查询和导出要有显式权限控制

## 日志与可观测性协同要求
- 高频慢查询要有定位手段
- 关键 SQL 风险点要能关联 traceId、模块和操作语义
- 不在日志中输出敏感查询结果全文

## 多租户适配要求
- 多租户项目必须统一 tenant 过滤策略
- 复杂报表、导出、分页查询必须验证 tenant 与数据权限未被绕开
- 平台级查询与租户级查询应显式区分方法和 SQL

## 文档要求
- 数据库文档要记录表、字段、索引、约束
- 模块文档要说明关键查询和写入规则
- 复杂 SQL、租户隔离策略、分页字段和排序字段白名单要有文档落点

## 常见误用
- 把查询参数和返回对象混用
- Service 层手写散乱 SQL 规则
- 用通用 Mapper 偷偷绕开租户或逻辑删除约束
- 不评估索引命中就直接上线复杂查询

## 禁止事项
- 在数据模型未稳定时过早沉淀复杂 Mapper 逻辑
- 在 schema、索引、查询 contract 未冻结时堆叠复杂持久化实现

## 追加增强：企业级数据访问细则

### 查询设计细则
- 列表查询必须显式区分筛选条件、排序字段、分页参数、权限范围
- 大查询优先考虑覆盖索引、条件裁剪、拆分查询或异步导出
- 报表型 SQL 与事务型 CRUD SQL 不应混在同一 Mapper 语义里

### 写入设计细则
- 批量更新、逻辑删除、状态流转必须显式校验影响行数
- 对乐观锁、生效状态、租户隔离字段的处理要统一

### Review 附加清单
- SQL 是否依赖未入库 schema
- explain 是否可接受
- 是否存在租户过滤、逻辑删除、数据权限绕过风险
