# 文件定位
创作者工作台 UI 规范包。

## 适用范围
- 创作者中心
- 内容协作平台
- 作品管理与发布工作台

## 典型技术组合位置
- 与 React、Next.js、Vue 3 等前端栈配合
- 适用于工作台、内容列表、详情编辑页、审核协作页和数据收益页

## 推荐目录结构
- 工作台、内容管理、编辑区、发布区、协作区和数据区有统一页面落点
- 创作工具区、侧边辅助区、状态区和协作反馈区边界清晰
- 列表、详情、编辑器承接页和数据页形成稳定页型

## 分层与职责边界
- UI 包定义工作台结构、内容页型、协作反馈模式和状态表达方式
- 不替代前端框架、编辑器实现和业务数据规则
- 视觉设计需兼顾创造力与操作效率，不能只追求“酷炫”

## 编码复杂度与可维护性要求
- 工作台、内容列表、详情/编辑、发布/审核和数据页的结构要稳定
- 创作工具区、资产预览区、协作信息区、状态区要显式分层
- 不允许在单页中堆叠过多互相竞争的操作重点

## 测试策略与质量门禁
- 内容创建、编辑、发布、审核、协作反馈和收益页关键路径必须覆盖
- 草稿态、审核态、发布态、失败态和权限受限态都要验证
- 浏览器级 smoke 应覆盖工作台首页、内容编辑承接页和发布链路

## 数据与 Migration 协同要求
- 内容状态、发布状态、审核状态、收益状态和统计字段必须与真实数据一致
- 状态或字段变更后，工作台页型和说明区要同步更新

## API / 安全 / 权限协同要求
- 创作者视角、审核视角、运营视角和结算视角的可见区块必须清晰
- 高风险发布、删除、审核和结算操作要有确认、权限和失败反馈

## 日志与可观测性协同要求
- 编辑失败、发布失败、审核失败、统计异常和收益异常要可定位
- 页面、作品、用户、状态和版本信息要可关联

## 多租户适配要求
- 多租户或多品牌创作者平台要明确空间切换、内容归属和资源隔离策略
- 不允许工作台状态与租户 / 空间上下文混乱

## 文档要求
- 记录工作台页型、状态表达、协作模式和高风险操作点
- 测试文档记录创作、审核、发布、收益的关键验证路径

## 常见误用
- 工作台像后台表格页，缺少创作导向
- 创作页、审核页、数据页风格断裂
- 状态太多但缺少主次和说明

## 禁止事项
- 在状态流和协作流未冻结前堆叠复杂视觉交互
- 只按视觉偏好打散工作台结构

## 追加增强：创作者工作台 UI 细则

### 工作台细则
- 首页应优先展示待办、作品状态、关键数据和快捷动作
- 创作入口、继续编辑入口和审核反馈入口要明确

### 编辑与协作细则
- 编辑承接页应区分主编辑区、辅助区、状态区和协作反馈区
- 审核意见、版本状态和发布反馈要有稳定落点

### Review 附加清单
- 是否存在工作台主次不清
- 是否存在编辑区与协作区互相抢焦点
- 是否存在状态表达与真实流程不一致
