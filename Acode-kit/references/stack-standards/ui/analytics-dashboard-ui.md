# 文件定位
数据分析看板 UI 规范包。

## 适用范围
- 指标驾驶舱
- 经营分析平台
- 数据看板与报表类前端项目

## 典型技术组合位置
- 与 React、Vue 3、图表库和数据聚合接口协同
- 适用于强筛选、强指标、强下钻的数据展示型页面

## 推荐目录结构
- 总览页、专题分析页、图表组件、筛选器组件和指标卡组件有统一落点
- 图表适配层、指标解释层、筛选器层和下钻详情层边界清晰
- 导出、异常提示、口径说明和刷新状态有稳定区块

## 分层与职责边界
- UI 包定义指标卡、图表编排、筛选器位置和看板页型
- 不替代具体图表库、前端框架和数据接口规范
- 图表视觉必须服务于分析效率，不追求无意义复杂装饰

## 编码复杂度与可维护性要求
- 图表类型、指标卡、筛选器、时间范围和维度控件要模块化
- 总览页与专题页的布局节奏保持一致
- 下钻入口、口径提示、异常提醒位置稳定
- 不允许图表、表格、筛选和说明区杂乱拼贴

## 测试策略与质量门禁
- 筛选、时间范围切换、图表渲染、下钻、导出和异常态必须覆盖
- 至少验证图表与明细、筛选与查询、口径提示与展示的一致性
- 浏览器级 smoke 需覆盖关键看板页和专题分析页

## 数据与 Migration 协同要求
- 指标口径、维度字典、时间粒度、导出字段与接口 contract 必须一致
- 字段或口径变化后，图表、指标卡、说明区和测试需同步更新

## API / 安全 / 权限协同要求
- 指标可见性、导出权限、敏感字段展示和平台 / 租户视角需统一受控
- 前端不伪造聚合结果或口径映射

## 日志与可观测性协同要求
- 图表渲染失败、关键查询失败、导出失败和空白看板要可定位
- 页面、查询条件、时间范围和版本信息要可关联

## 多租户适配要求
- 多租户看板必须区分平台视角、租户视角和组织视角
- 不允许不同租户的筛选缓存和查询结果串用

## 文档要求
- 记录看板页型、图表类型、指标解释、筛选器规则和下钻路径
- 测试文档记录关键分析链路和导出验证项

## 常见误用
- 图表很多但无主次
- 总览和专题页布局割裂
- 口径说明缺失导致用户误判

## 禁止事项
- 在指标定义未冻结时先堆复杂图表交互
- 用营销视觉模式直接处理数据看板

## 追加增强：分析看板 UI 细则

### 总览页细则
- 概览指标、趋势、异常和下钻入口必须形成稳定视线顺序
- 筛选器数量和位置应优先服务高频分析路径

### 图表与说明细则
- 图表旁必须预留口径说明、单位、时间粒度或异常说明位置
- 下钻入口和回退路径必须可理解

### Review 附加清单
- 是否存在图表与明细不一致
- 是否存在筛选和下钻链路混乱
- 是否存在指标解释缺失
