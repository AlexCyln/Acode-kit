# 文件定位
品牌展示 / 官网叙事 UI 规范包。

## 适用范围
- 企业官网
- 产品官网
- 品牌展示页
- 高叙事感营销 Web 项目

## 典型技术组合位置
- 与 Next.js、React、Astro 等 Web 前端栈配合
- 适用于强调品牌气质、视觉叙事和转化节奏的页面体系

## 推荐目录结构
- 首屏、价值段、案例段、CTA 段、FAQ 段采用稳定区块拆分
- 视觉资产、媒体背景、动效区块和文案区块有统一落点
- 页面模板、章节组件、转化组件边界清晰

## 分层与职责边界
- UI 包定义页面叙事结构、排版节奏、区块关系和视觉层级
- 不替代前端框架、动效引擎和通用组件库规则
- 视觉强化必须服务于价值表达与转化，而不是无意义装饰

## 编码复杂度与可维护性要求
- 区块式页面结构要稳定，可按 section 维护
- 首屏、卖点、证明、CTA 的层级关系清晰
- 大图、模糊背景、渐变、动效和字体策略需项目级一致
- 不允许每个页面各自发明新的叙事结构

## 测试策略与质量门禁
- 首屏、关键 CTA、表单留资、滚动叙事和响应式布局需覆盖
- 重点验证首屏加载、CTA 跳转、媒体资源加载和移动端表现
- 关键视觉段落至少有浏览器级 smoke

## 数据与 Migration 协同要求
- 页面文案、媒体资源、表单字段和埋点字段要与真实配置一致
- 若接 CMS 或表单后端，字段 contract 变更必须同步更新页面与文档

## API / 安全 / 权限协同要求
- 留资、预约、咨询表单必须统一错误反馈和成功回执
- 第三方表单、埋点、统计脚本的加载边界要清楚
- 不在页面代码中硬编码敏感 key 或生产回调地址

## 日志与可观测性协同要求
- 关键 CTA、留资成功 / 失败、资源加载失败和页面错误要可追踪
- 版本、页面、来源渠道和关键转化事件要可关联

## 多租户适配要求
- 多品牌或多站点项目需明确品牌切换、站点配置和资源隔离策略
- 不允许不同品牌主题和资产在构建产物中互相污染

## 文档要求
- 页面文档记录区块结构、关键叙事顺序、CTA 设计和媒体资源要求
- 测试文档记录首屏、CTA、表单和响应式重点验证项

## 常见误用
- 只追求“高级感”但信息层级混乱
- 首屏漂亮但价值表达不清
- CTA 多而散，叙事主链断裂

## 禁止事项
- 在叙事结构未冻结时过早堆叠复杂视觉装饰
- 用后台 UI 模式直接套官网展示页

## 追加增强：品牌展示 UI 细则

### 首屏细则
- Hero 必须同时承担品牌表达、价值表达和主 CTA
- 首屏图像、标题、辅助文案和 CTA 的层级要稳定

### 区块节奏细则
- 每个区块只承担一个主叙事目标
- 视觉高潮和转化区应有明显节奏控制，避免整页平均化

### Review 附加清单
- 是否存在视觉优先但信息表达失真
- 是否存在 CTA 路径分散
- 是否存在区块节奏和排版层级失控
