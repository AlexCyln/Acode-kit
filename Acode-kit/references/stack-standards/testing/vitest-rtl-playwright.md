# 文件定位
Vitest + React Testing Library + Playwright 前端测试规范包。

## 适用范围
- React Web 项目
- 需要覆盖组件交互、页面流程与关键端到端路径的前端项目

## 典型技术组合位置
- Vitest 负责快速单元/组件测试
- RTL 负责用户视角交互测试
- Playwright 负责关键流程 E2E

## 推荐目录结构
- 组件测试、页面测试、E2E 测试分层
- 测试夹具、mock 数据、辅助函数统一管理
- 登录上下文、权限态、字典 mock、路由夹具统一封装

## 分层与职责边界
- 组件测试验证局部交互和状态展示
- 页面测试验证查询、表单、权限和错误反馈
- E2E 验证跨页面关键链路、登录态、菜单和关键业务路径

## 编码复杂度与可维护性要求
- 以用户行为和可见结果命名测试
- 避免测试实现细节
- Mock 仅隔离不稳定外部依赖
- 测试逻辑也应通过夹具和 helper 降低重复与复杂度

## 测试策略与质量门禁
- 筛选、表单、弹窗、权限态、错误态要覆盖
- Step 5d 和 Step 5e 之间至少完成模块关键 UI smoke
- 列表页至少覆盖查询、筛选、分页、空态、错误态
- 表单页至少覆盖校验失败、成功提交、重复提交保护

## 数据与 Migration 协同要求
- 若页面依赖迁移后的新字段、新筛选项、新状态值，必须同步更新 mock、schema 和测试数据
- 不允许沿用过时字段继续通过测试假象掩盖真实变更

## API / 安全 / 权限协同要求
- 页面测试至少覆盖一个权限受限场景
- 关键页面必须验证接口失败反馈、无权限反馈、异常态回显
- 多角色或多租户页面至少覆盖一个越权受限场景

## 日志与可观测性协同要求
- 关键页面错误应能在浏览器预验收中捕获
- 页面级日志、埋点、错误上报的最小上下文要可验证

## 多租户适配要求
- 多租户前端至少验证上下文切换、列表隔离、详情隔离和入口权限差异

## 文档要求
- 测试文档记录关键页面路径、测试数据、失败复现方式和前置上下文
- 若页面依赖真实登录、菜单、权限或联调环境，需记录测试前置条件

## 常见误用
- 只测 happy path
- 只测组件不测关键页面流程
- 用快照测试替代交互和业务断言

## 禁止事项
- 在未确认批准设计状态前编写脱离设计的 UI 测试
- 要求用户人工验收前却未完成最低浏览器预验收

## 追加增强：前端测试执行细则

### 测试金字塔建议
- 组件测试解决局部交互和视觉状态
- 页面测试解决业务表单、筛选、权限和错误反馈
- Playwright 只覆盖核心闭环，不滥用到所有小改动

### 浏览器预验收细则
- 预验收必须覆盖真实菜单入口、关键按钮、失败反馈、控制台报错检查
- 若需要真实登录上下文，必须记录账号、角色和前置数据

### 评审附加清单
- 是否覆盖空态、失败态、权限态
- 是否验证用户能看见的最终结果，而不是内部实现细节
