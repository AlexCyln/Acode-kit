# 文件定位
JUnit + Mockito + Testcontainers 后端测试规范包。

## 适用范围
- Java / Spring Boot 项目
- 需要同时覆盖单元测试和关键集成测试的项目

## 典型技术组合位置
- JUnit 负责测试组织
- Mockito 负责边界隔离
- Testcontainers 负责真实依赖集成验证

## 推荐目录结构
- 单元测试与集成测试目录或命名明确区分
- 测试基类、测试数据工厂、容器初始化策略统一
- 数据库 / 缓存 / 中间件容器化测试可复用

## 分层与职责边界
- 单元测试优先验证 Service / Domain 规则
- 集成测试优先验证 Repository、API、事务、权限、多租户和 migration 链路
- Mock 用于隔离不稳定边界，不应掩盖真实 contract 问题

## 编码复杂度与可维护性要求
- 测试名必须体现业务行为
- 一个测试只验证一个主要断言主题
- 失败断言应能直接定位规则、状态或边界
- 测试代码也要控制复杂度，避免超长脚本式测试方法

## 测试策略与质量门禁
- Red-Green-Refactor 必须落实
- 核心事务、状态流、权限流、异常分支要覆盖
- Migration 至少要有 schema 演进验证或启动期迁移验证
- 关键导入导出、幂等、回调、多租户越权场景应有回归集

## 数据与 Migration 协同要求
- Testcontainers 环境要能复现关键数据库依赖
- Migration 脚本与当前代码版本一同验证
- 数据回填脚本要有迁移后核对点和重跑验证策略

## API / 安全 / 权限协同要求
- 关键 API 必须验证状态码、错误码、权限拦截、幂等行为
- 多租户和数据权限至少覆盖一个越权阻断用例

## 日志与可观测性协同要求
- 关键异常测试要验证日志和 trace 关键字段未丢失
- 关键异步或回调场景应验证最小观测数据可用

## 多租户适配要求
- 多租户项目至少验证单租户正常路径和跨租户越权阻断
- 平台级管理员与租户管理员权限差异要有测试样例

## 文档要求
- 测试文档记录关键用例、容器依赖和测试数据初始化方式
- 记录哪些测试属于单元、哪些属于集成、哪些覆盖 migration 和隔离风险

## 常见误用
- 全部用 Mock，跳过关键依赖集成验证
- 测试名称无法反映业务行为
- Testcontainers 只用来“跑起来”，却不做真实断言

## 禁止事项
- 先写实现再倒补测试
- 只跑 happy path 单测却声称模块可上线

## 追加增强：后端测试执行细则

### 测试分层建议
- 单元测试优先覆盖纯规则、状态机、校验、转换逻辑
- 集成测试优先覆盖 API、事务、Mapper、Migration、权限和租户隔离
- 回归测试优先覆盖历史高风险缺陷与线上事故复现链路

### 容器化测试细则
- 容器初始化要稳定、可复用、可本地与 CI 一致
- 测试数据初始化要可重复执行，不依赖手工状态

### 评审附加清单
- 是否真的覆盖了失败路径
- 是否把关键依赖全 mock 掉了
- 是否有 Migration 和权限链的实证测试
