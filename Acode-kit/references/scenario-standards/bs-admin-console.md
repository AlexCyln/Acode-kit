# 文件定位
面向 B 端后台管理台 / 控制台系统的场景规范包。

## 适用项目
- 企业内部后台
- SaaS 管理控制台
- 运营平台

## 典型业务目标
- 支撑配置、管理、审核、查询、运营和监控
- 提供高效率、低误操作的信息密集型后台

## 典型用户与角色
- 超级管理员
- 运营
- 审核人员
- 客服 / 支持人员

## 核心模块构成
- 仪表盘
- 列表 / 查询中心
- 详情页
- 配置页
- 权限 / 组织管理

## 页面/端形态特征
- 左侧导航 + 顶部信息区高频
- 表格、筛选、抽屉、弹窗、标签密集
- 多操作状态切换明显

## 数据与流程特征
- 查询、筛选、详情、编辑是主链
- 批量操作和审计日志常见

## 交付重点
- 信息层级清晰
- 列表效率
- 表单可理解性
- 风险操作保护

## 高风险点
- 筛选/列表效率差
- 弹窗过多导致流程割裂
- 关键操作无确认或无留痕

## 工作流附加约束
- Stage 2 先冻结导航框架、列表页模式、详情页模式
- Step 5b 要优先设计列表、详情、编辑三大基础页型

## 推荐配套技术栈方向
- React + shadcn
- B2B SaaS Admin UI

## 验收重点
- 常用管理链路高效
- 列表、详情、操作关系清楚
