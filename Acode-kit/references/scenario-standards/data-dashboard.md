# 文件定位
面向数据看板 / 经营分析 / 指标驾驶舱系统的场景规范包。

## 适用项目
- 经营分析平台
- 指标驾驶舱
- 数据看板与报表系统

## 典型业务目标
- 统一展示核心指标、趋势、维度拆解和异常提醒
- 帮助业务、运营、管理层进行分析和决策

## 典型用户与角色
- 经营管理层
- 数据分析师
- 运营负责人
- 区域 / 业务负责人

## 核心模块构成
- 总览驾驶舱
- 指标分析
- 趋势与维度拆解
- 报表中心
- 告警与异常追踪

## 页面/端形态特征
- 卡片、图表、筛选器、时间范围和对比视图高频
- 总览页 + 专题分析页 + 下钻详情页常见

## 数据与流程特征
- 聚合查询、维度筛选、时间序列和导出高频
- 指标口径、字典和刷新策略必须一致

## 交付重点
- 指标口径一致
- 信息层级清晰
- 维度筛选与下钻链路可理解

## 高风险点
- 指标定义不一致
- 图表与明细对不上
- 筛选条件过多导致使用效率差

## 工作流附加约束
- Stage 2 先冻结总览页、专题分析页和筛选交互模式
- Step 5c 优先确认指标口径、时间粒度、维度字典和导出字段
- Step 5d 强调图表、筛选、导出和异常态的测试先行

## 推荐配套技术栈方向
- Vue 3 / React 前端
- 强聚合查询或数仓接口
- 图表与报表能力分层

## 验收重点
- 指标、图表、明细一致
- 常用分析链路顺畅
- 下钻、筛选、导出可验证
