# 文件定位
面向创作者平台 / 内容协作平台的场景规范包。

## 适用项目
- 创作者中心
- 内容生产与协作平台
- 作品管理与分发后台

## 典型业务目标
- 支撑内容创建、编辑、发布、协作与收益结算
- 提供创作者视角和平台运营视角的协同能力

## 典型用户与角色
- 创作者
- 编辑 / 审核人员
- 运营
- 商务 / 结算人员

## 核心模块构成
- 创作工作台
- 内容管理
- 发布与分发
- 协作 / 评论 / 审核
- 数据与收益中心

## 页面/端形态特征
- 工作台、内容列表、编辑器承接页、详情页和统计页高频
- 作品状态、审核状态、分发状态和收益状态并存

## 数据与流程特征
- 内容对象、素材对象、发布对象和结算对象强关联
- 多状态流、多角色协作和历史留痕高频

## 交付重点
- 创作链路顺畅
- 状态流清晰
- 协作和审核边界明确

## 高风险点
- 内容状态与发布状态脱节
- 审核和协作链路责任不清
- 收益、数据和作品口径不一致

## 工作流附加约束
- Stage 2 先冻结工作台、内容列表、详情/编辑页和数据页骨架
- Step 5c 先明确内容状态流、审核流、发布流和收益口径
- Step 5d 强调编辑、发布、审核、收益展示链路测试先行

## 推荐配套技术栈方向
- React / Next.js 前端
- 内容服务 + 运营后台
- 对象存储、分发和统计能力协同

## 验收重点
- 核心创作与发布链路可用
- 状态、协作、审核和收益信息可追踪
