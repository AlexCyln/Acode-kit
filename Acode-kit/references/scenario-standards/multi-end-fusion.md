# 文件定位
面向多端融合项目的场景规范包。

## 适用项目
- 同时覆盖 Web、移动端、小程序的系统
- 管理台 + 用户端并存项目

## 典型业务目标
- 保证跨端能力统一，同时保留各端差异化体验

## 典型用户与角色
- 管理员
- 业务用户
- 终端用户

## 核心模块构成
- 跨端核心业务模块
- 各端差异化入口
- 统一账号与权限

## 页面/端形态特征
- 管理台偏信息密集
- 用户端偏任务/体验导向

## 数据与流程特征
- 核心实体统一
- 视图、交互、性能要求按端差异化

## 交付重点
- 核心业务规则一致
- 端间差异设计明确

## 高风险点
- 强行一套 UI 套全部端
- 端间状态定义不一致

## 工作流附加约束
- Stage 1 必须先冻结跨端共用能力与端专属能力
- Stage 2 必须先做总体多端信息架构，不允许直接做单端细稿

## 推荐配套技术栈方向
- 多端统一 API
- 端侧各自采用合适实现

## 验收重点
- 核心规则一致
- 各端体验符合各端场景
