# 文件定位
面向 B 端 SaaS 数据平台 / 数据分析运营平台的场景规范包。

## 适用项目
- 数据看板平台
- 指标平台
- 报表中心
- 数据运营分析系统

## 典型业务目标
- 汇总、分析、展示核心经营数据
- 提高指标口径统一性和数据消费效率

## 典型用户与角色
- 管理员
- 分析师
- 运营人员
- 业务负责人

## 核心模块构成
- 指标管理
- 报表 / 看板管理
- 查询筛选与导出
- 数据权限与订阅
- 任务调度 / 刷新记录

## 页面/端形态特征
- 看板首页、图表页、指标详情页高频
- 筛选器、日期区间、表格和图表并存

## 数据与流程特征
- 指标口径、维度、时间粒度是核心
- 查询链路和导出链路要求稳定

## 交付重点
- 指标定义一致
- 数据刷新状态明确
- 大表/大筛选场景可用

## 高风险点
- 口径不一致
- 图表展示与明细不一致
- 权限过滤造成数据误读

## 工作流附加约束
- Stage 3 必须先冻结指标实体、维度、聚合口径
- Step 5b 要明确筛选、钻取、导出状态

## 推荐配套技术栈方向
- 前端：React 管理台
- 后端：Spring Boot / 数据聚合服务
- UI：B2B SaaS Admin 风格

## 验收重点
- 核心报表口径一致
- 筛选结果可复核
- 看板与明细可追溯
