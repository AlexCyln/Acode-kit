# 文件名
`16_SECURITY_SPEC.md`

# 文件定位
认证鉴权、安全基线、敏感信息保护与审计规范。

# 适用范围
适用于前端、后端、数据库、缓存、部署配置、日志审计与 AI 生成代码的安全控制场景。

# 与其他文件的关系
本文件为 `04_BACKEND_ARCHITECTURE_SPEC.md`、`05_API_DESIGN_SPEC.md`、`07_REDIS_CACHE_SPEC.md`、`13_DEPLOYMENT_AND_DEVOPS_SPEC.md` 提供安全约束。

# 编写目的
统一认证鉴权、安全基线、输入校验、敏感信息保护与审计要求，防止项目在安全环节因实现差异产生漏洞。

## 1. 安全总原则
1. 默认最小权限。
2. 默认拒绝未经授权的访问。
3. 输入必校验，输出必脱敏。
4. 核心操作必审计。
5. 密钥与敏感数据不入代码仓库。

## 2. 认证与会话基线
1. 采用已声明认证方案（见 `PROJECT_OVERRIDES.md`）。
2. Access Token 生命周期应与风险等级匹配。
3. 登录成功需记录登录日志、设备信息、IP、时间。
4. 登出、强退、密码修改后应处理 Token 失效或黑名单。

## 3. 授权体系
1. 支持角色权限、菜单权限、按钮权限、接口权限、数据权限。
2. 权限码命名统一，如 `user:list`、`user:create`、`order:approve`。
3. 前端仅做展示拦截，后端必须做最终鉴权。
4. 数据权限至少预留“本人、本部门、本部门及子部门、全部、自定义”扩展点。

## 3.1 多租户与平台级权限
1. 多租户项目必须明确平台管理员、租户管理员、租户成员三类边界。
2. 平台级接口和租户内接口不得混用同一权限语义。
3. 租户上下文切换必须重新校验权限，不得沿用旧上下文缓存。

## 4. 输入校验
1. 所有外部输入必须经过参数校验。
2. 文件上传必须校验大小、类型、数量。
3. SQL、脚本、模板渲染输入必须做注入风险控制。
4. 对富文本、HTML 输入做 XSS 过滤或白名单清洗。

## 5. 敏感信息保护
1. 密码必须不可逆加密。
2. 手机号、邮箱、证件号、银行卡号、Token、密钥等按场景脱敏展示。
3. 日志中不得输出完整敏感字段。
4. 导出文件需评估是否脱敏。

## 6. 接口安全
1. 所有受保护接口必须校验 Token。
2. 幂等接口必须有防重复提交机制。
3. 关键操作需记录操作者、请求来源、结果。
4. 高风险接口可加验证码、二次确认或审批。

## 6.1 数据权限与导出安全
1. 列表查询、详情、导出、报表都必须受数据权限控制。
2. 导出接口必须复用后端最终权限和数据范围判断，不得只复用页面筛选条件。
3. 对跨租户、跨组织、敏感字段导出应记录审计并必要时强制二次确认。

## 7. 缓存安全
1. 验证码、登录态、幂等 Key 必须设置 TTL。
2. 缓存不存储长期唯一主业务数据。
3. 锁与会话 Key 命名应可追踪。

## 8. 日志与审计
1. 登录、登出、权限变更、数据导出、删除、审核、配置变更必须审计。
2. 审计日志应包含人、时间、动作、对象、结果、来源。
3. 审计日志不可被普通业务接口删除或篡改。

## 9. 安全编码禁令
1. 拼接 SQL。
2. 明文存储密码、密钥、Token。
3. 把权限判断仅放在前端。
4. 未校验文件直接落盘或执行。

## 10. 安全检查清单
1. 是否有统一认证。
2. 是否有后端最终鉴权。
3. 是否有输入校验。
4. 是否有敏感信息脱敏。
5. 是否有审计日志。
6. 是否有幂等与限流。
7. 是否有多租户/多组织隔离校验。
8. 是否对导出、回调、文件上传做边界保护。

## 本文件使用建议
所有涉及登录、权限、审核、导出、配置、部署的需求和代码变更均应引用本文件做安全自检与 Review。

## AI 调用建议
AI 在生成登录、权限、Token、导出、审计、上传相关代码时，必须优先遵守本文件的安全基线，不得为省事绕过鉴权和审计。

## 后续可扩展点
可增加 SSO 规范、OAuth2/OIDC 适配规范、数据脱敏策略矩阵、漏洞响应流程。

## 17. 企业级安全治理要求
1. 安全必须内建到需求、设计、编码、测试、部署和运维链路中，不得等到上线前统一补洞。
2. 身份认证、权限校验、数据访问控制、输入校验、审计记录、敏感数据保护必须作为默认能力存在，而不是按接口自选。
3. 后台系统和 SaaS 系统应默认假设存在越权、伪造请求、重放、批量滥用、配置泄露和跨租户访问风险。
4. 高风险动作如删除、导出、审批、配置变更、用户角色调整必须具备更严格的鉴权、确认和审计。

## 18. 敏感数据与边界保护
1. 密码、Token、证书、密钥、手机号、身份证号、银行卡号、地址等敏感信息必须分级保护和最小暴露。
2. 敏感数据不得出现在前端持久化存储、公开日志、异常堆栈、监控标签和调试截图中。
3. 外部回调、文件上传、导入导出、对象存储访问必须有类型校验、大小限制、权限校验和审计。
4. 多租户系统必须把租户越权视为高严重级安全问题，单独设计检测与回归验证。

## 19. Review 附加清单
1. 是否明确了认证、授权、审计、脱敏、限流、防重放和输入校验。
2. 是否存在高风险接口缺少二次确认、审计或权限隔离。
3. 是否评估了敏感配置、敏感字段和跨租户访问风险。
4. 是否能通过日志和审计记录复盘关键安全动作。
