# 文件名
`21_PROMPT_USAGE_GUIDE.md`

# 文件定位
让 AI 长期稳定遵循母规范的 Prompt 编写与上下文组织指南。

# 适用范围
适用于 Codex、VSCode AI、团队内部 Agent 在项目中持续生成需求、代码、SQL、测试与部署文件的场景。

# 与其他文件的关系
本文件与 `15_AI_COLLABORATION_PLAYBOOK.md` 配套使用，重点解决 Prompt 漂移、上下文不稳定、风格失控问题。

# 编写目的
统一 AI Prompt 的组织方式、规范引用方式和自检方式，让不同工具在长期协作中稳定输出符合母规范的结果。

## 1. Prompt 编写原则
1. 先给约束，再给任务，再给输入材料。
2. 明确指定引用哪些规范文件。
3. 明确输出物类型、语言、目录位置、禁止事项。
4. 对不确定项要求 AI 列出待确认项。

## 2. 推荐 Prompt 结构
```md
你必须遵守以下规范：
- global-engineering-standards/00_GLOBAL_ENGINEERING_PRINCIPLES.md
- global-engineering-standards/03_FRONTEND_ARCHITECTURE_SPEC.md
- global-engineering-standards/08_CODE_STYLE_AND_NAMING_SPEC.md

任务目标：
- 新增用户管理列表页与详情页

已知输入：
- Pencil 设计稿路径
- API 文档路径
- 项目级覆盖文档路径

输出要求：
- 生成 Vue 3 + TypeScript 代码
- 不得修改技术栈
- 先给待确认项，再实施
```

## 3. 不同场景的最小引用集合
1. 需求整理：`00`、`01`、`15`、`21`。
2. UI 实现：`00`、`02`、`03`、`08`、`15`、`21`。
3. 后端开发：`00`、`04`、`05`、`06`、`08`、`16`、`15`、`21`。
4. 部署与发布：`00`、`13`、`14`、`18`、`17`、`15`、`21`。

## 4. 降低上下文漂移的方法
1. 每次任务都重新声明引用文档。
2. 不把“默认规则”留给 AI 猜。
3. 把项目级覆盖单独列出。
4. 输出后要求 AI 自检“是否偏离固定技术栈”。

## 5. 常见反例
1. “帮我做个后台系统”这种没有约束和输入的 Prompt。
2. 只给截图，不给规范路径和输出要求。
3. 让 AI 自行选择前后端框架。
4. 让 AI 一次性生成整套系统但不给阶段边界。

## 6. 项目级 Prompt 资产建议
1. 固定系统 Prompt。
2. 阶段化 Prompt 模板。
3. 常用任务模板，如“生成列表页”“生成 CRUD API”“生成建表 SQL”。
4. 项目级术语表与权限码表。

## 7. AI 自检问题模板
```md
- 是否读取了本次任务要求的规范文件？
- 是否遵守了固定技术栈？
- 是否输出了待确认项？
- 是否遵守了命名、分层、接口、数据库规范？
```

## 8. 团队使用建议
1. 将常用 Prompt 模板版本化。
2. 在 PR 中附带本次 AI 使用的 Prompt 或摘要。
3. 对高风险任务保留“AI 输出 + 人工确认”双轨记录。

## 本文件使用建议
将本文件沉淀为团队标准 Prompt 模板库和 IDE Agent 指令库，减少每次临时说明导致的失真。

## AI 调用建议
AI 在接收任务时应先按本文件校验 Prompt 是否完整；若规范文件、输入材料、输出要求不全，应先补齐或显式声明假设。
